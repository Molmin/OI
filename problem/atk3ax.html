#atk3ax. Xor-Paths

时间限制：3 s       空间限制：250 MiB       标签： 搜索 双向搜索 暴力 位运算 CodeForces 

算法难度等级：5       思维难度等级：5       实现难度等级：5


本题来源于：Codeforces Round 498 (Div. 3) Problem F

题目大意

给定一个 n×mn\times mn×m 的地图 ai, ja_{i,\ j}a​i, j​​。

一条从左上角到右下角的路径，满足每次都只能向右或向下走，计算路径中经过的格子的 aaa 值的异或和。求这个和等于 kkk 的路径条数。

1≤n, m≤201\leq n,\ m\leq 201≤n, m≤20，0≤k≤10180\leq k\leq 10^{18}0≤k≤10​18​​

题目描述

There is a rectangular grid of size n×mn\times mn×m. Each cell has a number written on it; the number on the cell (i, j)(i,\ j)(i, j) is ai, ja_{i,\ j}a​i, j​​. Your task is to calculate the number of paths from the upper-left cell (1, 1)(1,\ 1)(1, 1) to the bottom-right cell (n, m)(n,\ m)(n, m) meeting the following constraints:

  • You can move to the right or to the bottom only. Formally, from the cell (i, j)(i,\ j)(i, j) you may move to the cell (i, j+1)(i,\ j+1)(i, j+1) or to the cell (i+1, j)(i+1,\ j)(i+1, j). The target cell can't be outside of the grid.

  • The xor of all the numbers on the path from the cell (1, 1)(1,\ 1)(1, 1) to the cell (n, m)(n,\ m)(n, m) must be equal to kkk (xor operation is the bitwise exclusive OR, it is represented as ^ in Java or C++ and xor in Pascal).

Find the number of such paths in the given grid.

输入格式

The first line of the input contains three integers n, mn,\ mn, m and kkk(1≤n, m≤201 \leq n,\ m\leq 201≤n, m≤20, 0≤k≤10180\leq k\leq 10^{18}0≤k≤10​18​​) — the height and the width of the grid, and the number kkk.

The next nnn lines contain mmm integers each, the jjj -th element in the iii -th line is ai,ja_{i, j}a​i,j​​(0≤ai,j≤10180 \leq a_{i, j} \leq 10^{18}0≤a​i,j​​≤10​18​​).

输出格式

Print one integer — the number of paths from (1, 1)(1,\ 1)(1, 1) to (n, m)(n,\ m)(n, m) with xor sum equal to kkk.
